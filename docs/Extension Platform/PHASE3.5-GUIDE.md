## 0. Orientation — 5 minutes

**Grain** is a local speech-to-text desktop app. Rust/Tauri backend in
`src-tauri/`, React frontend in `src/`, shared Rust crates in `crates/`, and a
separate native Rust window (the "pill") in `crates/grain-pill/`. It is built on
an upstream project called *Handy*, which lives verbatim in
`src-tauri/src/handy/` — **never add features there.**

**Extensions** add features without forking Grain. Three tiers: `pack` (data
only), `scripted` (JavaScript in a Web Worker), `native` (a separate program).
Every power is a **capability** declared in a manifest and enforced **in Rust**
on every message over a local WebSocket. Extensions never create windows — they
declare *surfaces* and the host builds, sleeps and destroys them.

**Phases 0–3 are shipped.** The runtime, the security wall, settings, and
extension-owned windows all work.

**The problem you are solving:** nobody outside the team can build an extension.
There is no CLI, no way to load one from a folder, no reload loop, no debugger.
That is why a real bug (C-1, a missing Tauri capability on surface windows)
survived all of Phase 3 — the only way to exercise a surface is to *be* an
extension author, and nobody could be one.

**Your acceptance test for the whole phase:** someone outside the team builds and
runs a Grain extension without asking a question.

### Facts about this repo you would otherwise waste an hour discovering

| Thing | Fact |
|---|---|
| Build the backend | Run `cargo` **from inside `src-tauri/`** — its `.cargo/config.toml` sets `target-dir = C:/gt`. A long target path makes whisper.cpp's CMake fail on Windows MAX_PATH. |
| Never do | Launch a second app instance while the user's Grain is running — port 7124, global shortcuts and the target-dir lock all collide. |
| Tests | `cargo test` in `src-tauri` (266), plus `crates/grain-sdk` (26), `crates/grain-pill` (23), `crates/grain-core` (18). Frontend: `npx tsc --noEmit`. All were green at the start of this phase. |
| `src/lib/bindings.ts` | **Generated by specta — never hand-edit.** New commands: use raw `invoke()` with a local mirror type until a dev run regenerates it. |
| `src/extension-runtime.ts` | The worker shim is a **backtick template literal**. Putting a backtick anywhere inside it — including in a comment — silently terminates the string. |
| Upstream ratchet | If and only if you touch `src-tauri/src/handy/`: commit first, then `python Upstream/ratchet.py`; on justified growth `--update` plus a follow-up commit. |
| Git | Commit each verified chunk promptly (main gets reset sometimes). **Never rebase** — on a rejected push use `git pull --no-rebase`, then push. Clean human commit messages, no AI attribution or `Co-authored-by`. |
| Tauri capabilities | A window with no matching file in `src-tauri/capabilities/` **cannot call `listen()`** — core APIs are permission-gated. App commands registered with `invoke_handler` are allowed everywhere by default. Window labels accept glob patterns. This asymmetry is exactly what hid C-1. |

## Step 1 — Validate `Origin` on the WebSocket handshake

**Why.** Browsers do **not** apply the same-origin policy to WebSockets: any web
page on the internet can open a connection to `ws://127.0.0.1:7124`. Our token
auth means it cannot *do* anything (classic cross-site WebSocket hijacking needs
ambient credentials, and we have none), but it can fingerprint that Grain is
running, and every pre-auth byte we parse is exposed to drive-by traffic. This is
correction **C-2**; see [DISTRIBUTION-RESEARCH.md](DISTRIBUTION-RESEARCH.md)
F-20.

**Where.** `src-tauri/src/events_server.rs`, in `async fn handle(...)` — it
currently calls `tokio_tungstenite::accept_async(stream)`.

**What to build.**

1. Switch to `accept_hdr_async` with a callback that reads the `origin` header.
   The crate is `tokio-tungstenite` **0.24**, where `ErrorResponse` is
   `http::Response<Option<String>>`; reject with `403 Forbidden`.
2. Put the decision in a **pure function** so it can be unit-tested without a
   socket:

   ```rust
   fn origin_allowed(origin: Option<&str>) -> bool
   ```

   | Origin | Verdict | Who that is |
   |---|---|---|
   | absent | **allow** | the pill and native-tier clients — Rust `tungstenite` sends no `Origin` |
   | `tauri://…` | **allow** | Tauri webviews on macOS/Linux |
   | `http`/`https` with host `tauri.localhost`, `localhost`, `127.0.0.1` or `::1` | **allow** | Tauri on Windows (`http://tauri.localhost`) and the dev server (`http://localhost:1420`, set as `devUrl` in `tauri.conf.json`) |
   | `null` | **reject** | a sandboxed iframe on someone else's page |
   | anything else | **reject** | every website on the internet |

   Handle a port suffix and bracketed IPv6 (`http://[::1]:1420`). Log rejections
   at `warn` **including the offending origin** — a wrong allowlist must be
   diagnosable from a user's log, not a mystery.
3. Add a cap on connections that have **not yet authenticated** (an
   `AtomicUsize` plus an RAII guard that decrements on drop and is released once
   `identity` is established, around line 186). 64 is far above what Grain needs:
   the pill plus one per activating extension.

**Do not** break these two real clients: `src/extension-runtime.ts` (runs in a
blob-URL Worker created by the `extension-host` page, so it inherits that page's
origin) and `src/extension-surface.ts` (the surface wrapper page).

**Done means.** Unit tests cover every row of the table above, including `null`
and a real-looking `https://evil.example`. The app boots and the log still shows
`events WS: 'pill' authenticated`.

## Step 2 — `grain-ext init`

**Why.** The first five minutes decide whether someone builds a second
extension. Anthropic's MCPB toolchain is two verbs (`init`, `pack`) and that is
the right level of cleverness (F-13).

**Where.** A new crate, `crates/grain-ext-cli/`, producing a binary named
`grain-ext`. Rust, because tiers A and C need no Node at all.

**What to build.** `grain-ext init <name>` scaffolds a manifest, an entry file, a
README, a `.gitignore`, and **TypeScript types generated from `grain-sdk`** so
the API has exactly one source of truth. Tell the author up front, in `init`'s
output, that a *scripted* extension needs Node + esbuild for bundling — Raycast
requires the same, and discovering it later is worse.

**Done means.** A fresh scaffold passes `grain-ext doctor` (step 8) with zero
findings.

## Step 3 — Load unpacked

**Why.** This is the "run my code" primitive everything else hangs off.

**Where.** Registry: `crates/grain-core/src/extensions.rs`. Host:
`src-tauri/src/extension_host.rs`. UI:
`src/components/settings/experimentations/`.

**What to build.** An Extensions ▸ **Developer** section, hidden until a
**developer-mode toggle** is switched on, with a **folder picker** ("Load
unpacked…"). Loaded extensions get the `dev` trust rung and a permanent badge.
If a store version of the same id is installed, show the store one as
**"Overridden by dev extension"** — Zed's wording, because the ambiguity is
otherwise a genuine time sink (F-17).

**Security rules, not preferences (F-10).** Developer mode is switched on only by
a human, in-app. It is never enabled by a link, a download, or an extension. A
dev extension gets the **identical capability wall**, can never display as
verified, and can never be promoted. While developer mode is on, a persistent
chip is visible in the Extensions tab.

**Done means.** A scaffolded extension loads from disk, appears in Overview
badged `dev`, and a capability it did not declare is denied exactly as it would
be for an installed extension.

## Step 4 — `grain-ext dev` and hot reload

**Why.** VS Code still cannot reload an extension without restarting its host,
and it is their best-known DX complaint. We can, because a scripted extension
**is** a Web Worker and the host already kills and respawns workers on every idle
reap.

**Where.** `src-tauri/src/extension_host.rs` (worker lifecycle),
`src/extension-host.ts` (the JS supervisor), `src-tauri/src/events_server.rs`
(token minting — see `mint_extension_token`).

**What to build.** The CLI watches files → rebuilds → pushes over a **dev
channel** on the existing local WebSocket → the host kills the worker, respawns
it, and re-mounts any surface iframes. The dev token lives in a `0600` file in
the app data directory and exists **only while developer mode is on**. Identity
stays bound to the channel — the platform's existing rule — and the dev token
grants **dev-control methods only**, never extension capabilities.

**Done means.** An edit is live in **under 300 ms** with no app restart, and RAM
after ten reloads equals RAM after one. Measure the second one; a leaked worker
per reload is the obvious failure and it is invisible without a measurement.

## Step 5 — Source maps

**Why.** Today a scripted extension's JavaScript is a string embedded in the
manifest, so a stack trace points at `entry_source:1`. That is correction
**C-6**.

**What to build.** Dev builds emit a source map; the host maps thrown stacks back
to the author's files before they reach the developer panel.

**Done means.** A deliberate `throw` reports the author's file and line.

## Step 6 — The developer panel

**Why.** This is where the phase is won or lost. Raycast's dev mode shows errors
with full stack traces *in the UI the developer is already looking at*, and it is
the single most-praised thing about their platform (F-16).

**Where.** Build it as a **workspace surface** using the existing host-owned
generic in `src-tauri/src/surfaces/workspace.rs`, so it sleeps like everything
else. **Build it only when developer mode is on** — an unused window is a
violation of this project's core rule.

**What to show, live:** activation events and worker spawn/reap · every host call
with method, params, result and **duration** · **denials in red, naming the
missing capability and the exact manifest line to add** · transform timings
against the 150 ms budget with p50/p95 and strikes · worker memory against its
ceiling · console output and errors, source-mapped. Plus one **"Copy
diagnostics"** button.

**Done means.** An intentional capability denial appears within a second, naming
the capability and the fix. With developer mode **off**, no panel window exists
and idle RAM is unchanged — measure both.

## Step 7 — Typed errors

**Why.** Zed returns an error naming the missing capability, and it is the
highest-leverage decision in their extension API (F-18). A silent empty result is
the difference between five minutes and an afternoon.

**Where.** Error codes in `crates/grain-sdk/`; call sites in
`src-tauri/src/host_api.rs`.

**What to build.** `E_CAPABILITY_DENIED`, `E_TIMEOUT`, `E_QUOTA`,
`E_INVALID_MANIFEST`, … each carrying `message`, `hint` and `docs`. Then sweep
the host API for paths that currently return an empty success on refusal.

**Done means.** A test asserts no host-API path returns a bare empty success when
it refused to do something.

## Step 8 — `grain-ext doctor`

**Why.** Round-trips, not review, are what make publishing feel slow. `doctor`
must run **the identical code** CI will later run, so a submission passes first
time (DISTRIBUTION-PLAN §3.4).

**What to build.** Manifest validation, capability-name checks, size caps,
activation sanity, budget lint — and **rejection of invisible and bidirectional
Unicode** (correction **C-4**). That last one is not theoretical: the GlassWorm
campaign hid payloads in exactly those codepoints so reviewers' editors rendered
nothing (F-6).

Put the lint in a **shared crate** so Phase 5B's CI consumes the same code rather
than a reimplementation that drifts.

**Done means.** A source file containing a zero-width character fails with the
file, the line, and the offending codepoint.

**What to build.** A quickstart, the capability reference, three worked examples

**Done means.** Someone outside the team builds and runs an extension **without

## Step 10 — Verify C-1 for real

**Why.** During the Phase 3.5 design pass, `src-tauri/capabilities/extension-surface.json`
was added because `ext-surface-*` and `ext-overlay-*` windows matched no
capability file, which would have denied `listen()` and broken the sleep/revive
handshake. **The fix has never been exercised at runtime**, because doing so
requires an extension with a surface — which is what step 3 finally makes
possible.

**What to do.** With a dev extension that declares a workspace surface: open it,
sleep it, revive it, and deliver a payload. Confirm sleeping returns to baseline
RAM.

**Done means.** Phase 3's surface handshake is proven end to end, closing the gap
that hid the bug in the first place.
